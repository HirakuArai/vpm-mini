
啓の仕事マップ（Draft）

本ドキュメントは、啓個人が現在担っている主な仕事を棚卸しし、
「仕組みに任せうる部分」と「本人にしか向き合えない部分」をラフに仕分けすることを目的とする。

このマップは、vpm-mini においてどのタスクをセル化・仕組み化していくかを判断するための足がかりになる。

1. 主な仕事カテゴリ（例）

定例レポート・定期KPI分析

臨時の深掘り分析（特定テーマの調査・仮説検証）

意思決定支援（上長や関係者との議論・選択肢提示）

資料作成・プレゼンテーション

AI/仕組みづくりそのもの（vpm-mini 等）

※ 実際の中身は今後のセッションで具体化する。現時点ではラフなカテゴリ列挙とする。

2. 任せうる部分 vs 任せにくい部分（観点）

任せうる部分の例：

データ抽出・整形・基本集計

定型フォーマットのレポート生成

過去のパターンに基づく「おおよその見立て」の提示 など

任せにくい部分の例：

関係者の文脈や感情を踏まえた最終判断

啓自身の価値観や人生観に強く依存する選択

長期的な方向性や「何を大事にするか」を決める場面 など

この区分は固定的なものではなく、AIと仕組みの進化に応じて変化しうる前提とする。

4. 今後の使い方

新しくタスクが増えたときや、既存タスクの性質が変わったときには、このマップを更新する。

persist-report や他のセル群が増えてきたときに、
「どのタスクがどのセルにどの程度任せられるようになったか」を振り返る基盤として使う。
